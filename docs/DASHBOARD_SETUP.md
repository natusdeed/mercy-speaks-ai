# Mercy AI Dashboard — authentication setup

The dashboard is **private**: there is **no public signup**. A single **work email** and **shared password** are stored as **server-only environment variables**. After login, the app stores a short-lived **HMAC-signed session token** (not a Supabase Auth user).

## What you are configuring

| Variable | Purpose |
|----------|---------|
| `MERCY_DASHBOARD_ADMIN_EMAIL` | Only this email may sign in (must match what you type on the form; stored value is compared case-insensitively). |
| `MERCY_DASHBOARD_PASSWORD` | Shared secret password for that admin (generated by the bootstrap script). |
| `MERCY_DASHBOARD_SESSION_SECRET` | Server secret used to sign and verify session tokens. |

**Optional alias:** you may set `MERCY_DASHBOARD_ADMIN_PASSWORD` instead of `MERCY_DASHBOARD_PASSWORD` (same meaning). Prefer `MERCY_DASHBOARD_PASSWORD` in docs and Vercel.

## One-command setup (recommended)

From the **repository root** (`mercyspeaks.ai`):

```bash
npm run dashboard:bootstrap -- your@email.com
```

The script prints three lines. Copy them exactly.

### Local development

1. Create or edit **`my-app/.env.local`** (this file is gitignored).
2. Paste the three variables from the bootstrap output.
3. **Restart** the dev server (`npm run dev`). Vite only reads env files at startup.

Env files are loaded from **both** the repo root and `my-app/` (see `my-app/vite.config.ts`), so **`my-app/.env.local`** or **repo-root `.env.local`** both work; `my-app/.env.local` keeps dashboard secrets next to the app.

### Vercel (production / preview)

1. [Vercel Dashboard](https://vercel.com/) → your project → **Settings** → **Environment Variables**.
2. Add each variable name with the same values you use locally (use **Production**; add **Preview** too if preview deployments should access the dashboard).
3. **Redeploy** so new variables apply to serverless `/api/dashboard/*` routes.

## What to enter on the login page

- **Email:** exactly the address you passed to bootstrap (e.g. `natusdeed@gmail.com`) — it must match `MERCY_DASHBOARD_ADMIN_EMAIL`.
- **Password:** the value of `MERCY_DASHBOARD_PASSWORD` from your env (or from the bootstrap output). Copy/paste is best; the string is long and random.

## Rotating credentials

Run bootstrap again with the same or a new email. Update **every** place the old values lived (local `.env.local`, Vercel, any other host). Changing `MERCY_DASHBOARD_SESSION_SECRET` invalidates existing sessions immediately.

## Troubleshooting

- **“Dashboard sign-in is not configured”** — one or more of the three variables are missing or empty on the **server** that handles `POST /api/dashboard/login`. Fix env and restart/redeploy.
- **Works locally but not on Vercel** — set the same three variables in the Vercel project and redeploy; confirm they are enabled for the right environment (Production vs Preview).
